# Game Design Document — Project Dusk (MVP)
**Жанр:** Single-player RPG с открытым миром, вид от первого лица (ASCII/text)  
**Вдохновение:** TES2: Daggerfall — масштаб, свобода, процедурная генерация  
**Платформа:** Windows/Linux, Python 3.11+, терминал  
**Целевая версия MVP:** 0.1.0

---

## 1. Концепция

Игрок оказывается в процедурно сгенерированном мире — материке с городами, подземельями и дикой природой. Никакой единой главной сюжетной линии в MVP; акцент на систему выживания, исследование и ролевые механики.

---

## 2. Основные системы MVP

### 2.1 Мир
- **Карта мира:** сетка регионов (16×16 клеток), каждый регион — тип местности (лес, горы, равнина, побережье, пустыня)
- **Регионы содержат:** 1 город, 0–3 деревни, 1–4 подземелья, случайных NPC на дорогах
- **Процедурная генерация:** сид-based, воспроизводимый мир по числу

### 2.2 Персонаж
- **Расы:** Человек, Эльф, Орк (пассивные бонусы к атрибутам)
- **Классы:** Воин, Маг, Разбойник (стартовые статы + 2 способности)
- **Атрибуты (8 шт.):** Strength, Intelligence, Willpower, Agility, Speed, Endurance, Personality, Luck
- **Производные:** HP, Mana, Stamina, Carry Weight, Movement Speed
- **Навыки (12 шт.):** Blade, Blunt, Archery, Block, Sneak, Lockpick, Alchemy, Destruction, Restoration, Illusion, Speechcraft, Mercantile
- **Прокачка навыков:** через использование (use-based leveling, как в TES)
- **Уровень:** повышается каждые N очков навыков, дают +5 к 3 атрибутам на выбор

### 2.3 Боевая система
- **Тип:** пошаговая, текстовая
- **Механика:** skill-based, без броска на попадание — удар всегда достигает цели, урон зависит от навыка и силы атакующего минус броня/ловкость защищающегося (как Skyrim). Ключевое решение — выбор действия, а не результат кубика
- **Действия за ход:** Атака, Силовой удар (x2 урон, тратит Stamina), Блок (поqвышает митигацию), Уклон (повышает митигацию через скорость), Использовать предмет, Бегство (успех зависит от разницы Speed)
- **Stamina:** тратится на силовые удары, восстанавливается каждый ход
- **Статусы:** Bleeding, Poisoned, Stunned
- **Смерть:** загрузка последнего сохранения

### 2.4 Инвентарь и предметы
- **Категории:** оружие (клинок, дробящее, лук), броня (6 слотов), расходники (зелья, еда), ключи, деньги (gold)
- **Вес:** у каждого предмета вес, ограничение по Carry Weight
- **Прочность:** вещи изнашиваются, требуют ремонта у кузнеца
- **Магические предметы:** простые аффиксы (bonus damage, resist fire и т.д.)

### 2.5 NPC и диалоги
- **NPC-типы:** торговец, стражник, житель, квестодатель, враг
- **Диалог-система:** ключевые слова (как в Daggerfall) — игрок выбирает тему из списка
- **Репутация:** глобальная (0–100) влияет на цены, квесты, агрессию

### 2.7 Квесты
- **Квесты MVP:** процедурно генерируемые (fetch, kill, escort, deliver)
- **Квест-борд:** в каждом городе, 3–5 квестов, обновляются каждые 7 игровых дней
- **Награда:** gold + репутация + случайный предмет

### 2.8 Время и отдых
- **Игровые сутки:** 24 часа, время идёт при ходьбе и действиях
- **Отдых:** в таверне (стоит gold) или лагере (риск нападения)
- **Голод/усталость:** опциональная система (флаг в настройках)

### 2.9 Экономика
- **Торговля:** buy/sell у торговцев, цены зависят от Mercantile + Personality
- **Инфляция:** нет (MVP)
- **Хранилище:** сундук в купленном доме (дом покупается в городе)

## 3. Интерфейс (TUI)

Используем библиотеку `curses` (или `tcod` как fallback). Экраны:

| Экран | Описание |
|-------|----------|
| Main Menu | Новая игра / Загрузить / Выход |
| Character Creation | Раса → Класс → Атрибуты → Имя |
| World Map | 16×16 ASCII карта с иконками регионов, курсор |
| Location View | ASCII-арт города/подземелья, список POI |
| Combat | Лог боя + статусная строка + меню действий |
| Inventory | Список предметов, drag-and-drop слоты экипировки |
| Dialogue | Имя NPC + ответы NPC + кнопки тем |
| Status | Атрибуты, навыки, уровень, XP |
| Journal | Активные/завершённые квесты |

---

## 4. Сохранения

- JSON-файлы в `saves/` директории
- Автосохранение при входе/выходе из локации
- Слот быстрого сохранения (F5) + 3 ручных слота

## 5. Архитектура (высокий уровень)

```
RPG/
├── main.py              # точка входа, инициализация curses
├── engine/
│   ├── game.py          # GameState, главный игровой цикл
│   ├── world.py         # процедурная генерация мира
│   ├── combat.py        # боевая система
│   ├── events.py        # event bus
│   └── time.py          # игровое время
├── entities/
│   ├── character.py     # базовый класс персонажа
│   ├── player.py        # игрок (наследник character)
│   ├── npc.py           # NPC
│   ├── item.py          # предмет
│   └── spell.py         # заклинание
├── ui/
│   ├── renderer.py      # curses-рендерер
│   ├── screens/         # по одному файлу на экран
│   └── widgets.py       # переиспользуемые UI-элементы
├── data/
│   ├── races.json
│   ├── classes.json
│   ├── items.json
│   ├── spells.json
│   └── npc_names.json
├── saves/               # игровые сохранения
├── tests/
└── GDD.md
```

## 6. Out-of-scope для MVP

- Графика (sprites, 3D)
- Магия (школы заклинаний, мана)
- Мультиплеер
- Мод-поддержка
- Политическая система / фракции (выше репутации)
- Транспорт (лошади, корабли)
- Крафт
- Недвижимость / строительство
- Permadeath

---

## 7. Метрики успеха MVP

- [ ] Можно создать персонажа и пройти первый квест от начала до конца
- [ ] Мир генерируется за < 2 сек
- [ ] Сохранение/загрузка работают без потери данных
- [ ] Боевая система проходима без крашей
- [ ] Работает на Windows и Linux без доп. зависимостей (кроме Python std + `tcod`)
